Créer un Commit via l'Invite de Commandes

  • Auteur : ACOMDIR https://www.facebook.com/acomdir/.

Un commit enregistre les modifications apportées à votre projet. C'est une étape clé dans la gestion des versions d'un projet avec Git.

Étape 1 : Ajouter des fichiers au suivi Git

Avant de créer un commit, vous devez ajouter les fichiers modifiés ou nouveaux au suivi de Git :

git add .

Cette commande ajoute tous les fichiers modifiés ou nouveaux dans le répertoire actuel.

Pour ajouter un fichier spécifique :

git add nom_du_fichier

Étape 2 : Créer un Commit

Une fois les fichiers ajoutés, créez un commit avec un message décrivant les modifications :

git commit -m "Message décrivant vos modifications"

Exemple :

git commit -m "Ajout du fichier README.md et correction de bugs mineurs"

Étape 3 : Vérifier le Commit

Pour vérifier que votre commit a été créé avec succès, utilisez :

git log

Cette commande affiche l'historique des commits, y compris l'identifiant, la date et le message de chaque commit.

Étape 4 : Pousser les Modifications vers GitHub

Pour synchroniser vos commits locaux avec le dépôt distant, utilisez :

git push origin main

Remplacez main par le nom de votre branche si nécessaire.

Conseils Supplémentaires

  • Écrivez des messages de commit clairs et concis pour décrire vos modifications.
  • Utilisez des commits fréquents pour sauvegarder votre travail étape par étape.
  • Vérifiez toujours l'état de votre dépôt avec git status avant de créer un commit.
